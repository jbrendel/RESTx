What is a resource?

A resource is a something that can be directly used, without requiring any further knowledge. Usually, it provides a link that can just be clicked on to get some data. Often, it is also possible to perform updates to the underlying data, using nothing but ordinary web requests to send the information.

A resource is created by providing a set of parameters for the components, which are specialized, generic pieces of code living in the RESTx server, which implement a very specific function, such as accessing a database, a cloud service, a proprietary in-house API, etc. Components can also take advantage of RESTx and implement even complex data integration logic in just a few lines of code.

Once these parameters are provided, RESTx stores them for later use and creates the URL of a new resource. Once that URL is accessed, the parameters are loaded again and applied to the component.
