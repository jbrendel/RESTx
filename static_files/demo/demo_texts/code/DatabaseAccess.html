DatabaseAccess: A use case for specialized components

The DatabaseAccess component implements access to an SQL database. To set up database access, important, security sensitive or quite technical information needs to be provided as configuration. For example, the database name, the JDBC driver class or the connection string.

Using specialized components to pre-configure and hide fields

Often it is NOT desirable to share this information. At the same time, we want to enable users of the system to create their own RESTful web services and resources based on the database access we configure for them.

To facilitate this, RESTx uses the concept of "specialized components". In effect, these are partially configured resources. For example, the IT staff could create specialized component, based on the DatabaseAccess component: Only the DB connection string and other critical information is provided. This is then stored as a new 'specialized component'. Users then can create new resources based on this specialized component: They still can configure their own fields and where clauses, but do not have the ability to change the DB connection string or JDBC driver class.
