A component example: Google Search

The GsearchComponent implements access to Google's search API. It will serve as an example to show us how to use components and how to create resources based on them.

Component meta information

RESTx shows a lot of useful information about a component. Scroll around in the frame below. You can see a short, human readable description of the component, its name and even a link to the component's built-in documentation.

A component is generic, reusable code. To make it work for you, first a configuration has to be provided for it. When such a set of parameters has been presented to the server, a RESTful resource is created (please see here for more information about RESTful resources). Therefore, it is very important that the component tells us which parameters it requires in order to create a resource. It does so right here with its meta information, where the params section tells us about these parameters. In addition to the component specific parameters there are also a few additional parameters, which always need to be provided, independent of the type of component. For all parameters you can see a short description, a type, a flag indicating whether this parameter is required or optional and a default value (for optional parameters).

Scroll down a little more and you come to the services section. A 'service' in RESTx is the name of a sub-resource. It is implemented in the component by a function/method of the same name. Our GsearchComponent implements a single service method, called search. You can see that again we get a description, a list of parameters, but also a list of acceptable input and output content types. RESTx takes care of content negotiation and all necessary transformations. Adding custom content types is easy.

By displaying this information, RESTx makes it easy to determine if the provided component is of interest to the client. Please remember that the entire API description here is automatically generated by RESTx after examining the component code. The component author merely declares those parameters and service methods in the code. You can see examples of how this is done for Java, Python or server-side JavaScript. All three languages can be used to write components.

Next:

We are now ready to create our first RESTful resource. To do so, we need to provide some parameters for this component. This can be easily done either via the RESTful API or simply by filling out a form in right here in the browser.

To see the form, please scroll all the way to the top and click the Create resource link.
