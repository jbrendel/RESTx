Conclusion

You have now seen how easy it is to crate a new RESTful web service with RESTx. Some of the key points:

  • The new service hides even complex configuration details behind a simple, easy to use URI. Users of this new service don't need to concern themselves with any of the implementation details.
  • New RESTful resources and web services can be created easily and without programming, merely by filling out a form in a web browser. This allows users to create their own RESTful resources and web services, without always having to involve back-end server administrators or developers.
  • RESTx automatically produces a descriptive, easy to use RESTful API for components and resources.
  • Components can be easily written in several different programming languages.
