A new RESTful resource, ready to use

Just like RESTx showed us meta data about components, it also shows similar information about a resource. At first glance this looks quite similar. However, please notice that the parameters you provided during resource creation time are not shown anymore. This makes perfect sense, since users of the new resource do not have to concern themselves with the technical details of what's behind that resource.

As before, you can see a description and the name of the resource.

Below that you can see the description of the available services again, with all the usual details, including description of input and output types as well as parameters. That way, a client has all the information they need in order to use the services (or sub-resources) provided by this resouce.

Finally, as part of the service description, we get a link which when clicked invokes the service method of the component, which implements this sub-resource.
