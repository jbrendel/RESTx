Welcome to the RESTx interactive demo!

An online demo showing the simple creation of RESTful web services and self-documenting RESTful APIs

What is RESTx?

RESTx is a light-weight, open-source platform for the creation of RESTful web services, data access and integration resources. It emphasizes simplicity, sane defaults and out-of-the-box usability. No complex configuration, no steep learning curve. After a small, compact download you will be up and running in less than 5 minutes. Find out more on the RESTx project web site.

Is it just another web application framework?

No! RESTx is specialized in the rapid creation of RESTful web services and resources. It differs from traditional application frameworks in that code is organized in reusable components. These components are specialized pieces of code, providing one particular function such as database access, integration with a custom API, access to an SaaS service, etc. Writing custom components is very easy and can be done in either Java, Python or server-side JavaScript.

RESTful web services and resources can be created simply by providing new parameter sets to those components. This demo will show you exactly how this is done and how simple it is. You can also go to the RESTx web site and find out more about how RESTx is different.

What will I see during this demo?

Actually, this is more than just a demo: You get to actually drive a live RESTx server and see how easy it is to use. All the while, we will tell you about what you are seeing and how you can use the information returned by the RESTx server.

How will the demo / walk-through work?

We will split this page into three frames. In the bottom frame you see the pages exactly as they are returned by the RESTx server. In the middle frame you see the URI from which that page was fetched. In the top frame, we will tell you about the page you are seeing. We might give you some instructions on where to click next. If you follow them you will truly get a guided tour of the RESTx server. It's that simple.
